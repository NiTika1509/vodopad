Требования по созданию системы бронировая номеров в сети отелей:
1. Стек разработки:
Vue 3
Vuex
Vue Router
PrimeVue
Laravel
MySQL
Nginx
Docker

2. Реализация REST API, разработка api с обработкой ошибкой и уведомлением
   пользователя об ошибках.

3. Связка с бд FireBase

4. Система должна иметь функцию регистрации для обычных пользователей и авторизации


5. Система должна иметь ролевую модель в виде: Администратор и Пользователь-Арендатор


6. Администратор должен быть в ручную создан в базе



7. Навигация у Арендатора должна быть: Отели, Мои заказы


8. Навигация у Администратора должна быть: Отели, Обработка заказов


9. На странице Отели пользатель системы должен увидеть фильтры по поиску номеров,
   на странице должна быть возможность смена вида отображения выбора номера в отеле,
   отображение выбранного отеля и физуальное отображение расположение номеров
   относительно отеля (пример: в отеле пять этажей, по три номера на каждом этаже,
   отображение отеля дожно быть ввиде прямоугольника, с пятью рядами, по три в каждом,
   квадратика)
   9.1. На странице должна быть возможность отображения выбора номера как в виде
        визуального выбора номера относительно отеля, так и ввиде списка номеров
   9.2. При выборе визульного отображения отеля, должно быть визуальное отображение цветом
        свободных и занятых номеров
   9.3. При наведении на квадрат номера должно быть всплывающее окно с информацией о номере
   9.4. При нажатии на квадрат номера пользователя должно перенаправить на страницу номера
	  с более подробной информацией о номере
   9.5. На странице номера у Арендатора должна быть кнопка с должна быть возможность
        забранировать номер.


10. На странице Мои заказы Арендатор должен увидеть все свои заказы, в том числе и
    архивные, с актуальным статусом заказа для новых заказов.


11. На странице Обработка заказов Администратор должен иметь возможность увидеть
    актуальные заказы от пользователей системы и обработать их (изменить статус заказа)
    11.1. Статусы заказы должны быть: в обработке, согласовано, отказано, архивный
    11.2. До обработки администратором все новые заказы имеют статус в обработке
    11.3. Статус согласовано присваивается в случае успешной обработки заказа
    11.4. Статус отказано присваиваются заказу в случае каких-либо проблем с номером
    11.5. В случае отказа Администратор должен в свободной форме указать причину заказа
	    что в последствии должен будет увидеть у себя в заказах Арендатор
    11.6. Статус архив присваивается заказам после того, как Арендатор после аренды номера
          успешно съехал из него
